Spectral Shipped

Modular Python WiFi auditing framework built for BlackArch Linux. Designed for network security researchers — clean CLI interface, pluggable modules, and minimal dependencies. It grew out of frustration with existing tools that were either bloated with a GUI nobody needed or locked behind a licence. Spectral stays at the command line, ships as separate modules so you only load what you're testing, and runs comfortably on low-resource hardware like a Raspberry Pi.

Game Booster In Progress

Android performance optimization app using Shizuku for ADB-level system access — no root required. Built with Kotlin and Jetpack Compose for a clean, modern UI. It reads live CPU, RAM, and thermal data straight off the device and lets you kill background processes and clear cached memory without root — something most "booster" apps on the Play Store fake or simply can't do. The UI is built around one idea: show real numbers, not vague progress bars.

Phone Footprint Audit Tool Shipped

Python OSINT script that analyses the digital footprint of a phone number — aggregates data across public APIs and directories to build a comprehensive profile. It cross-references number formatting, carrier lookup, and public directory listings, then compiles everything into a single readable report instead of forcing you to check five different sites by hand. Built for a fast first-pass lookup before digging in manually.

About

Who I Am

I'm Rana Ahmad Umar — a self-taught developer from Lahore, Pakistan, coding since 2022. At 21, I specialise in building things that actually work: Python & Bash automation scripts, AI-integrated tooling, Android apps with Kotlin and Jetpack Compose, and Linux/BlackArch security utilities.

My background leans technical and OSINT-adjacent — I enjoy digging into systems, automating the tedious, and building CLI tools that professionals actually want to use. Available for freelance engagements worldwide.

I didn't come up through a CS degree or a bootcamp — everything here was learned by breaking things, reading documentation late at night, and rebuilding a project for the third time until it finally made sense. That path is slower in some ways, but it means I've had to genuinely understand why something works, not just copy a pattern that happens to compile.

On a freelance job, that translates into fewer surprises: I ask the annoying clarifying questions up front, give realistic timelines instead of optimistic ones, and would rather tell you a feature isn't worth building than quietly pad an invoice. If something breaks after delivery, I fix it — that's part of the job, not a separate line item.
